## 4. Contracts an incoming agent must honor

### Terminal run record (`terminal_runs.build_run_record`)
Keys: `run_id, timestamp, actor, command, cwd, classification, policy_decision, allowed, approval_required, exit_code, stdout_excerpt, stderr_excerpt, duration_ms, redactions_applied, output_truncated, trust_state="Tier 4", terminal_output_trusted=False, audit_paths{json,markdown}`. Persisted under `07_LOGS/Terminal-Runs/<YYYY-MM-DD>/<run_id>.json|.md`.

### Session export result (`exports.export_session`)
Returns `{ok, session_id, format, export_path, artifact_manifest_path, audit_path, redaction_applied, redaction_report, external_upload_performed=False}`. Session records live at `07_LOGS/Chaser-Sessions/<session_id>.json`.

### Studio terminal workbench (`StudioAPI.get_terminal_workbench(command="", limit=20)`)
Returns `_ok` envelope wrapping `{policy, preview, recent_runs, audit, run_detail_contract, authority{shell_execution_from_studio:False,â€¦}, cli_contract, warnings}`. Studio NEVER executes.

### Studio terminal run detail (`StudioAPI.get_terminal_run_detail(run_id)`)
Returns `_ok` envelope wrapping a read-only detail result from `terminal_runs.load_terminal_run_detail()`. Unsafe ids and missing runs return compact `ok:false` error shapes. Terminal output remains `terminal_output_trusted:false` and Tier 4.

### CLI
`chaseos operate terminal policy|preview <CMD>|run <CMD> [--cwd --timeout --max-output --actor]|history [--limit]|show <RUN_ID>`. `--json` on all. `run` exits 3 if blocked, 0 on success, 1 on nonzero/error. `show` reads existing audit records only and exits 2 for unsafe/missing ids.

## 5. Non-negotiable governance (applies to every unit above)

1. No unrestricted terminal, no sudo/elevation, no shell operators (`&& || | ; > <` `$(` backtick) â€” all blocked at classification.
2. Terminal/tool/session output is **Tier 4 untrusted**; it can be displayed/searched/cited but is never an instruction and never mutates canonical truth.
3. No model-provider calls from Studio, the gateway, the board, or ChaserAgent core. Provider dispatch goes Bus â†’ runtime; the runtime owns credentials ([[Provider-Agnostic-Routing-Architecture]]).
4. No hidden external upload (session export sets `external_upload_performed=False` and means it).
5. No silent host mutation â€” startup items / cron / services / scheduled tasks only via the existing runtime-gateway-controls approval path.
6. ChaserAgent / Chaser Gateway do not bypass the ChaseOS Gate, Permission-Matrix, or Trust-Tiers. Session ids are routing selectors, not auth tokens.
7. Studio cannot expand a permission ceiling. The backend/CLI cannot bypass core policy.
