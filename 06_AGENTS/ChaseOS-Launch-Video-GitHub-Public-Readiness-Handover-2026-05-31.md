# ChaseOS Launch Video + GitHub/Public Readiness Handover

**Date:** 2026-05-31  
**Primary domain:** `chaseos.systems`  
**Purpose:** Define what the launch videos, GitHub/public repo, public docs, website story, and distribution assets must show — and what they must not expose or overclaim.

---

## 0. Core launch story

The public launch should prove one thing:

```text
ChaseOS turns scattered AI work into one governed knowledge graph where projects, sources, agents, runtimes, workflows, approvals, and outputs stay connected.
```

Do not make the video about “another AI chat app.”  
Do not make it only about startup validation.  
Do not make it about reckless autonomy.  
Do not make it about a marketplace that does not yet exist.

The launch story is:

```text
AI-native builders are already using ChatGPT, Claude, Codex, local docs, repos, browser workflows, agents, and half-finished automations.
The pain is scattered context, lost memory, unsafe execution, and no governed operating layer.
ChaseOS gives them a local-first operating system: memory, graph, sources, runtimes, approvals, missions, Forge packs, and safe writeback.
```

## 9. Required GitHub files

### README.md must include

```text
- what ChaseOS is
- who it is for
- what is ready
- what is partial/future
- install/run instructions
- local-first/privacy statement
- chaseos.systems link
- waitlist link
- Forge preview link
- standards/docs link
- license/open-core explanation
```

### SECURITY.md must include

```text
- how to report security issues privately
- no public exploit details requested
- supported versions
- expected response posture if known
- what not to include in public issues
```

### CONTRIBUTING.md must include

```text
- contribution scope
- pack submission rules
- no secret capture
- no hidden browser automation
- no protected-core bypasses
- license expectations
- review process
```

### LICENSE / open-core language

Be precise:

```text
Open-source means OSI-compliant.
Source-available means people can inspect code but rights are restricted.
Commercial/proprietary means paid or restricted use.
```

Do not call commercially restricted code “open-source.”
