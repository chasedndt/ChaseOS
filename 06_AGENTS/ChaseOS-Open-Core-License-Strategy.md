# ChaseOS Open-Core License Strategy

## Terms

- **True open-source:** OSI-compatible license, reuse/modification/distribution permitted under the license.
- **Source-available:** code viewable but commercial/reuse rights restricted.
- **Commercial source:** customer-visible/source-escrow or licensed source with commercial restrictions.
- **Proprietary paid layer:** closed or licensed service/product capability.

Do not claim commercially restricted code is open-source.

## Open-source candidates

Public standards, JSON schemas, manifest examples, validators if intended, SDK examples, free sample packs, public docs, and compatibility examples.

## Source-available / open-core candidates

Basic local ChaseOS Core, basic Studio, trust-sensitive runtime/governance code if leadership chooses this, Forge install protocol, AOR/Gate contracts, and Chaser Agent base harness if released.

## Commercial / paid candidates

Hosted accounts, runtime credits, managed agents, premium Studio features, premium workflow packs, team features, enterprise/private deployment, payment/licensing infrastructure, private Forge registries, support, and managed reliability.

## Creator pack classes

- Free packs: 0% take-rate.
- Paid creator packs: 9% future marketplace strategy.
- Certified packs: 12–15% future strategy.
- Managed/runtime packs: 15–20% plus runtime credits future strategy.
- Enterprise/private packs: custom.

## Codex implementation/legal task list

Codex should prepare or update `LICENSE`, `NOTICE`, `CONTRIBUTING.md`, `SECURITY.md`, marketplace terms, pack creator terms, privacy policy, and terms of use. Hermes/reviewer must verify that language distinguishes open-source from source-available/commercial layers.
