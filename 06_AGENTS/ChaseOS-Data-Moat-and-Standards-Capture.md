# ChaseOS Data Moat and Standards Capture

## Data moat

ChaseOS data moat is not extraction of private graphs. It is the user-owned continuity created by local private graph memory, source/project/decision context, approval/runtime records, reusable workflow outcomes, and eventually opt-in anonymized aggregate benchmarks and Forge ecosystem signals.

Public-safe wording:

> Your private graph stays yours. ChaseOS improves for you locally. If you opt in, anonymous workflow outcomes may help improve packs, recommendations, and reliability for everyone.

## Standards capture

ChaseOS should define portable formats for:

- workflow pack manifests
- agent runtime contracts
- approval packets
- graph nodes/edges
- source evidence records
- mission outputs
- outcome records
- Forge index entries
- entitlements/licensing later
- managed job specs later
