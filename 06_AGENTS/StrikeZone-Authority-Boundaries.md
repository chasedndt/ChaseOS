# StrikeZone Authority Boundaries

Links: [[StrikeZone-Workflow-Pack]] · [[StrikeZone-Crypto-OS]] · [[HERMES]] · [[Hermes-Runtime-Profile]] · [[Agent-Activity-Index]]

Status: DRAFT / BOUNDED LOCAL AUTHORITY

## Allowed

- Read declared StrikeZone vault/project files.
- Use the operator-confirmed logged-in Chrome CDP profile for read-only evidence capture.
- Capture screenshots/rendered text from X, Grok, Perplexity, TradingView, CoinAnk, and YouTube subscription pages.
- Rebuild evidence manifests and validation reports.
- Generate local operator briefs and Discord-review markdown drafts.
- Generate prediction candidates for operator review.
- Append only operator-approved candidates to local scorecard JSONL.
- Record later backtest/resolution events.

## Forbidden

- No trade execution.
- No wallet, exchange, broker, or signing actions.
- No automatic public/member Discord posting.
- No X likes, follows, posts, DMs, deletes, or settings changes.
- No Grok/Perplexity account mutation beyond reading existing operator-positioned pages.
- No YouTube likes/comments/subscriptions or settings changes.
- No CoinAnk/TradingView settings/subscription/payment/trial actions.
- No credential/cookie/token extraction.
- No canonical knowledge promotion from prediction outcomes.
- No scorecard-to-trade automation.

## Approval rule

The workflow may generate drafts and private status summaries. External delivery and scorecard submission require explicit operator approval. Trade execution is outside this workflow entirely.
