## Selection Principles

A viable ChaseOS mark should communicate:

- human core
- agent network
- private boundary
- controlled execution

It should feel:

- calm
- premium
- technical
- sovereign
- private
- intelligent
- controlled

It should avoid:

- generic AI sparkles
- literal robots, brains, locks, or shields
- crowded node diagrams
- crypto/Web3 visual language
- cyberpunk clutter
- copying Claude, OpenAI, Obsidian, Linear, Vercel, Raycast, or Cursor
- mascot-led primary identity
- founder-personal symbolism as the main software logo

## Recommended Exploration Set

### Direction 1: Sovereign Core

Recommendation: `PRIMARY EXPLORATION`

Concept:
A minimal system glyph built from a central operator core, a restrained C-shaped privacy shell, and implied permissioned agent geometry.

Visual structure:

- central core for human intent
- partial C-shaped boundary for ChaseOS, command, containment, and privacy
- two to four small controlled nodes for agent runtime and memory
- slight rotational or execution-path implication without looking like a space mark

Why it fits:

- It directly carries the source brief's recommended sentence: a subtle C-shaped sovereign shell containing a central operator core and implied permissioned agent graph.
- It can scale from app icon to product mark.
- It expresses privacy and control without using a literal lock.
- It can become a UI motif for approvals, runtime state, and graph memory.

Risks:

- Too many nodes make it look like a generic graph logo.
- Too obvious a C makes it feel like a plain monogram.
- Too much orbit motion makes it feel decorative instead of operational.

Best use:

- primary ChaseOS mark
- Studio app icon
- website hero mark
- favicon/app icon exploration

Prompt seed:

```text
Design a premium minimal logo for ChaseOS, a privacy-first agentic operating system for human-agent collaboration. Create an original abstract system glyph: a subtle C-shaped sovereign shell containing a central operator core and a small implied permissioned agent graph. Dark-first, calm, technical, premium, private, controlled. Use restrained obsidian, graphite, runtime teal, human amber, and bone-white accents. Avoid robots, brains, locks, sparkles, crowded network diagrams, crypto aesthetics, cyberpunk clutter, and imitation of existing AI or notes-app logos.
```

### Direction 2: Command Kernel

Recommendation: `SECONDARY EXPLORATION`

Concept:
A sharper operator/developer mark that fuses command cursor energy with kernel/core geometry and a subtle OS glyph.

Visual structure:

- central kernel point or cutout
- command-line/cursor suggestion, kept abstract
- compact OS-style geometry
- minimal C or bracket-like containment

Why it fits:

- It emphasizes execution, runtime control, and command-layer positioning.
- It fits technical founders, developers, AI engineers, and operator audiences.
- It can pair well with terminal, CLI, Agent Bus, and runtime surfaces.

Risks:

- If too terminal-like, it narrows ChaseOS to developer tooling.
- If too angular, it can lose the calm premium tone.

Best use:

- secondary route if Sovereign Core feels too abstract
- developer-facing docs
- CLI/runtime identity motif
- optional Studio submark

Prompt seed:

```text
Create an original ChaseOS logo direction called Command Kernel. It should feel like a private command layer for human-agent execution: a compact OS/kernel glyph with subtle C geometry, central operator core, and restrained command-cursor influence. Calm, premium, technical, dark-first, controlled. Avoid making it a terminal prompt icon, generic devtool logo, robot, brain, lock, sparkle, crypto mark, or copied AI-brand geometry.
```

### Direction 3: Permission Matrix

Recommendation: `UI MOTIF / TERTIARY LOGO EXPLORATION`

Concept:
A bounded modular grid or matrix with a central human-controlled node and explicit routed paths to agent nodes.

Visual structure:

- small grid or matrix frame
- central core
- two to three controlled paths
- boundary emphasis
- enough asymmetry to avoid looking like a spreadsheet or enterprise grid

Why it fits:

- It maps directly to ChaseOS permission matrices, trust tiers, runtime profiles, and bounded execution.
- It can become strong UI language for approvals and agent scopes.

Risks:

- As a primary logo it may feel too literal or operational.
- Grid marks can look generic if the form is not highly custom.

Best use:

- Approval Center icon language
- permission-gate states
- runtime profile visuals
- system diagrams
- possible secondary mark

Prompt seed:

```text
Explore a ChaseOS Permission Matrix mark: a premium abstract bounded matrix with a central human operator node and two or three permissioned runtime paths. It should suggest secure execution, trust tiers, and agent governance without becoming a literal grid dashboard. Dark-first, minimal, original, calm, technical. Avoid crowded node diagrams, literal locks, shields, robots, sparkles, and generic enterprise grid logos.
```

### Direction 4: Quiet Wordmark + System Glyph

Recommendation: `WORDMARK STUDY`

Concept:
A restrained ChaseOS wordmark with a small custom glyph embedded near the C or OS, using typography as the primary brand signal.

Visual structure:

- exact canonical wordmark: `ChaseOS`
- custom OS or C treatment
- optional small symbol for app icon derivation
- no loud illustration

Why it fits:

- It makes the canonical name unambiguous.
- It can feel mature and product-grade.
- It avoids overloading the first logo with too many concepts.

Risks:

- Wordmark-only direction may not provide enough icon identity.
- Needs careful font licensing and custom vector work before final use.

Best use:

- website header
- docs
- investor/customer material
- pairing with Sovereign Core as symbol-only mark

Prompt seed:

```text
Create a premium custom wordmark study for ChaseOS. The typography should be calm, precise, technical, and product-grade, with a subtle custom treatment around the C or OS to imply operating system, command, and intelligence. Canonical text must be ChaseOS. Avoid gamer, cyberpunk, playful, generic SaaS, overly futuristic, or copied typography.
```

## Selection Checklist

A candidate should pass these tests:

- Meaning: does it express human core, agent network, private boundary, controlled execution?
- Scale: does it work at 16px, 32px, app icon, and hero size?
- Originality: does it avoid looking like existing AI, notes, devtool, or crypto brands?
- Product credibility: would it sit beside serious software products without feeling gimmicky?
- System expansion: can it become UI states, loading marks, approval motifs, graph states, and iconography?
- Naming: does it preserve canonical `ChaseOS` spelling?
- Boundary: does it avoid implying uncontrolled AI autonomy, surveillance, or broad host control?
