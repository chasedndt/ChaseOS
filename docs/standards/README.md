# ChaseOS Standards

ChaseOS standards define the file formats, manifests, contracts, and protocols that make workflow packs, Forge listings, agent runtimes, approval packets, graph objects, source evidence, outcomes, entitlements, and managed jobs portable across the ChaseOS ecosystem.

## Draft standards

- `chaseos.pack.json` — workflow pack manifest.
- `chaseos.forge-index.json` — public Forge catalog index.
- `chaseos.agent.json` — runtime capability/permission contract.
- `chaseos.approval.json` — approval packet format.
- `chaseos.graph.json` — knowledge graph object format.
- `chaseos.source.json` — source provenance object.
- `chaseos.outcome.json` — workflow outcome record.
- `chaseos.entitlement.json` — future licensing/entitlement object.
- `chaseos.managed_job.json` — future managed-runtime job packet.

Standards are early and must not be presented as stable API guarantees until validators and compatibility tests exist.
