# SOUL.md — Public Placeholder

> This public repository must not contain a real personal or private SOUL.md.
> Use `SOUL.template.md` to create your own private instance-local `SOUL.md` after cloning or forking ChaseOS Core.

---

## Purpose

`SOUL.md` is the identity and operating-character layer for a private ChaseOS instance. In a real deployment, it may describe the operator's identity, values, voice, active domains, decision posture, and session protocols.

Because that information is personal context, the public Core repository keeps this file as a neutral placeholder only.

---

## How To Use

1. Copy `SOUL.template.md` to `SOUL.md` in your private instance.
2. Populate the private copy with your own identity, doctrine, voice, risk posture, and active projects.
3. Keep the populated `SOUL.md` out of public repositories unless you intentionally want to publish that personal context.
4. Store operational rules in `00_HOME/Assistant-Contract.md`, not in `SOUL.md`.
5. Store tool/runtime routing in files such as `CLAUDE.md`, `AGENTS.md`, or runtime-specific docs, not in `SOUL.md`.

---

## Public Core Boundary

This placeholder may define the role of `SOUL.md`, but it must not contain:

- A real person's name, location, biography, or private identity details
- Personal projects, private goals, or life context
- Financial, trading, career, health, family, or location context
- Credentials, tokens, keys, account details, or private endpoints
- Runtime-specific private memory or deployment state
- Claims that bind a fork to the original operator's personal instance

The authoritative public framework reference is `SOUL.template.md`.
